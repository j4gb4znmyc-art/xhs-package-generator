# 得物图文种草内容工作流生成器

这是一个本地可运行、也可部署到 Streamlit Cloud 的「得物图文种草内容工作流生成器」。

工具不直接生成图片，不接入任何图片 API，也不需要 OpenAI API Key。用户只需要填写产品名和品牌名，工具会自动识别品类、生成同品类差异化卖点，并输出可复制给 ChatGPT 继续执行的图文工作流。

## 必填字段

- `product_name`：产品名
- `brand_name`：品牌名

## 选填字段

- 产品图
- 风格选择：`ALL` / 单选 / 多选
- 输出组数：默认 10 组，最多 10 组
- 核心卖点补充
- 功效依据补充
- 品类、使用场景、目标人群
- 禁用词和额外要求

## 功能

- 只填产品名和品牌名即可生成
- 自动识别产品品类
- 自动生成不少于 8 条同品类差异化卖点
- 区分通用体验卖点和需要依据支持的功效卖点
- 自动筛选 5-8 条适合当前产品的核心卖点
- 支持 1 到 10 组风格输出
- 每组包含 5 张独立图片策划和 1 篇种草文章
- 每组都生成可单独复制给 ChatGPT 的执行提示词
- 支持产品图上传和本地预览
- 支持 `jpg/jpeg/png/webp/heic/heif`
- 支持 Markdown / JSON 导出

## 输出结构

```text
# 得物图文提示词生成结果
## A. 产品基础信息
## B. 同品类差异化卖点库
## C. 本次筛选出的核心卖点
## D. 本次风格队列
## E. 分组执行包
```

每个分组包含：

- 本组风格定位
- 本组 5 张图策划
- 本组文章
- 可直接复制给 ChatGPT 的本组执行提示词

## 本地运行

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

打开：

```text
http://localhost:8501
```

## Streamlit Cloud 部署

仓库根目录需要包含：

```text
app.py
requirements.txt
README.md
```

部署参数：

```text
Branch: main
Main file path: app.py
```

继续推送到同一个 GitHub 仓库后，Streamlit Cloud 会自动更新，公网网址保持不变。
